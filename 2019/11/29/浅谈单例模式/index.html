浅谈单例模式

发表于 2019-11-29 | 分类于 设计模式 | 评论数：

应用

我最初了解单例模式的时候是大一的时候用javafx写即时聊天系统时，为了解决跨线程之间的通信。后来才知道这就是单例模式啊。
所以单例模式应用场景可以解决跨线程的通信、或者像安卓中生命周期最长的 Application作为框架的全局数据存储地方。

单例模式很简单，但是却隐藏着新手很难发觉的危险。下面将分析不同的写法的优缺点。

写法一

分析

背景：在并发的环境下且instance未初始化
发生事件：两个线程同时执行到 if(instance == null)

思考：

  • 两个线程下一步分别执行哪一行代码？
  • instance = new Single();执行了几次？

在分析之前，可以看看这种写法有没有解决写法一中的问题。

思考：

  • 有几个线程能同时执行if(instance == null)？（或者两个线程能不能同时获取instance实例）
